## 验收标准（逐条）

### P0 — 阻断性问题

- [ ] **V1**：未登录访问任何页面 → 自动跳转登录页
- [ ] **V2**：注册 → 登录 → 进入首页，流程无断点
- [ ] **V6**：完整二创流程跑通
  - 登录 → 输入链接 → 预览 → 选风格 → 生成 → 结果 → 历史记录
  - 所有环节无异常，与登录前行为一致

### P1 — 核心功能

- [ ] **V3**：用户 A 创建的任务，用户 B 的历史记录里看不到
- [ ] **V4**：第一个注册的用户，数据库 role 字段为 admin
- [ ] **V5**：用户可以在设置页填写 Cookie，检测通过后保存

### P2 — 边界情况

- [ ] 已登录状态刷新页面 → 不跳登录页
- [ ] 注销 → 跳登录页 → Token 被清除
- [ ] 未配置 Cookie 时创建任务 → 提示配置
- [ ] 旧任务（user_id=NULL）不报错
- [ ] Docker Compose 全部重建无报错

## 建议执行顺序

1. `docker compose build && docker compose up -d` 全部重建
2. 清除浏览器 localStorage，打开新标签页
3. **V1**：直接访问首页 → 应跳登录页
4. **V2+V4**：注册第一个用户 testadmin → 确认 role=admin → 登录 → 进入首页
5. **V5**：进入设置页 → 填写 Cookie → 保存 → 检测有效性
6. **V6**：完整二创流程（用一个真实小红书链接测试）
7. 注销 → 注册第二个用户 testuser → 确认 role=user
8. **V3**：用 testuser 登录 → 历史记录看不到 testadmin 的任务
9. 修复发现的问题
10. Git commit 全部改动
11. 最终 rebuild + deploy
